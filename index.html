01
Account Universe Definition
ICP → structured TAM · not a lead list
Agent +

What happens here

  • AI Agent (Claude) reads ICP criteria document → outputs structured Apollo query parameters
  • Firmographic filters: company size 500–5000, industry vertical, HQ region, revenue signals
  • Technographic filters: CRM in use, key platform signals from Apollo + job post keywords
  • LinkedIn headcount growth filter — companies scaling >15% YoY prioritised
  • Output: scored account list, not contacts. Accounts first — always.

Why accounts before contacts

  • Enterprise deals are won or lost at the account level — not the contact level
  • Prevents researching individuals at accounts that don't fit
  • Forces ICP discipline before any outreach motion begins
  • Account scoring gates the entire downstream pipeline
# AI Agent (Claude) — ICP to Apollo query parameters
import subprocess, json

def build_account_universe(icp_doc: str) -> dict:
    result = subprocess.run([
        "claude", "-p",
        f"Read this ICP document and output a JSON Apollo search query.\n"
        f"Include: industries[], company_size_range, keywords[], tech_stack_signals[].\n"
        f"JSON only, no explanation.\n\nICP:\n{icp_doc}"
    ], capture_output=True, text=True)

    query = json.loads(result.stdout)
    # → passes to Apollo account pull
    return query

Committee roles mapped per account

  • Economic Buyer — CFO, VP Finance, budget owner. Needs ROI framing, risk reduction.
  • Technical Evaluator — CTO, VP Engineering, IT. Needs integration depth, security, architecture.
  • Champion — Director/Senior IC who benefits most. Needs career win angle.
  • End User — Ops, RevOps, Sales. Needs ease of use, efficiency gain.
  • Blocker — Legal, Compliance, Procurement. Needs risk mitigation, vendor credibility.

What AI Agent (Claude) does

  • Takes company name + org size → maps likely committee structure
  • Cross-references Apollo contacts against role taxonomy
  • Tags each contact with role, influence level, and sequence angle
  • Flags accounts with missing key roles — don't sequence until Champion identified
  • Output: committee map JSON per account, feeds outreach architecture

# AI Agent (Claude) — signal research + structured brief
def build_signal_brief(account: dict, job_posts: list, headcount_data: dict) -> dict:
    context = {
        "company": account["name"],
        "headcount_growth": headcount_data["growth_pct"],
        "active_roles": [j["title"] for j in job_posts],
        "funding": account.get("last_funding", "unknown")
    }
    result = subprocess.run([
        "claude", "-p",
        f"Build a GTM signal brief for this account.\n"
        f"Extract: primary_trigger, hook_angle, timing_urgency (1-10), tech_gaps[].\n"
        f"JSON only.\n\nContext:\n{json.dumps(context)}"
    ], capture_output=True, text=True)
    return json.loads(result.stdout)

04
Multi-Score Qualification Gate
Three scores · positive + negative weights · hard disqualifiers
Agent Critical Gate +

Three-score model

  • Fit Score — firmographic match. Threshold ≥60.
  • Intent Score — behavioural signals. Threshold ≥40.
  • Timing Score — urgency trigger. Threshold ≥30.
  • All three must pass. One failure = account parked, not discarded.
  • Parked accounts re-enter scoring queue when new signal fires.

Negative weights — what decreases score

  • Company >10K employees → Fit −20 (likely locked vendor)
  • No detectable tech stack → Fit −15
  • Hiring freeze signals → Intent −25
  • Recent major implementation → Timing −20
  • No CTO/CIO in seat → Timing −15
  • Active M&A target → Timing −20
# Scoring engine — positive + negative weights + hard disqualifiers
def score_account(account: dict, signals: dict) -> dict:
    fit, intent, timing = 0, 0, 0

    # Fit — positive
    if 200 <= account["headcount"] <= 5000: fit += 25
    if account["industry"] in TARGET_INDUSTRIES: fit += 20
    if signals["has_crm"]: fit += 15
    if account["funding_stage"] in ["Series B", "Series C", "Series D"]: fit += 20

    # Fit — negative
    if account["headcount"] > 10000: fit -= 20
    if not signals["tech_stack_detected"]: fit -= 15

    # Hard disqualify — exits immediately
    if signals["is_competitor_customer"]: return {"qualified": False, "reason": "competitor_lock"}
    if signals["recently_renewed_competitor"]: return {"qualified": False, "reason": "contract_locked"}

    qualified = fit >= 60 and intent >= 40 and timing >= 30
    return {"fit": fit, "intent": intent, "timing": timing, "qualified": qualified}

Multi-Thread Outreach Architecture
One sequence per role · coordinated waves · signal-grounded copy
Agent +

Role-specific messaging logic

  • Economic Buyer — ROI framing, risk reduction, board narrative. No feature talk.
  • Technical Evaluator — integration depth, security posture, architecture fit. No buzzwords.
  • Champion — career win angle, internal visibility, makes them the hero. Personal tone.
  • End User — efficiency, less manual work, faster outcomes. Concrete before/after.
  • Sequences are timed to create internal discussion — not fired simultaneously

Sequence structure — 4 touches

  • Day 1 — Signal hook per role. References specific trigger from brief.
  • Day 4 — Value add. Relevant insight for their function.
  • Day 8 — Social proof. Outcome relevant to their industry + role.
  • Day 14 — Break-up. Short, no hard sell. Leaves door open.
  • Champion sequence starts Day 1. Economic Buyer starts Day 3 — after Champion is warm.

What gets populated before rep touches it

  • Export format: structured JSON → map to target CRM field schema
  • Human review step before import — 15 min per 50 accounts

Feedback Loop — Intelligence Compounds
Every interaction improves the system · scoring recalibrates weekly
Agent +

What the agent ingests weekly

  • Reply data — which signals generated responses, which didn't
  • Meeting outcomes — which committee roles converted to meetings
  • CRM deal notes — objections, competitive mentions, deal context
  • Lost deal reasons — feeds competitive intelligence layer
  • Sequence engagement by role — open rates, click-throughs, reply sentiment

What the agent outputs

  • Scoring weight recommendations — increase/decrease signal weights
  • Segment performance report — which ICP profiles convert best
  • Copy iteration flags — sequences underperforming vs baseline
  • Competitive intelligence update — new objections, new competitors cited
  • Parked account triggers — re-score accounts that now meet timing threshold

Where This Breaks

System Limitations

Signal Layer
Apollo + LinkedIn data has lag
Job post data is typically 2–4 weeks behind. LinkedIn headcount is self-reported. First 2–3 runs require manual brief validation before trusting signal automation. Calibrate weights on real reply data, not assumptions.
Committee Mapping
Org structure isn't always visible
For companies with thin LinkedIn presence, committee mapping relies on job post inference. Don't sequence if Champion is unidentified — a sequence without a Champion is just cold outreach with extra steps.
Generation Layer
AI-assisted copy still needs human review (~10%)
Hallucinated context, wrong seniority, tone mismatch — rare but real. The human review step at export isn't optional. 15 minutes per 50 accounts is the actual cost of running this safely.
Scale
Built for precision, not volume
This system targets 30–100 high-fit accounts per month, not thousands. Enterprise pipeline quality requires depth of intelligence per account. If you need volume outbound, this is not the right architecture.
